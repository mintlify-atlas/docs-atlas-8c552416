Andrej Karpathy Skills is a single-file set of behavioral guidelines that address the most common LLM coding pitfalls — derived from [Andrej Karpathy's observations](https://x.com/karpathy/status/2015883857489522876) on how language models go wrong when writing code. Install it as a Claude Code plugin, a Cursor project rule, or a per-project `CLAUDE.md` file.

## The problem these guidelines solve

From Andrej Karpathy:

> "The models make wrong assumptions on your behalf and just run along with them without checking. They don't manage their confusion, don't seek clarifications, don't surface inconsistencies, don't present tradeoffs, don't push back when they should."

> "They really like to overcomplicate code and APIs, bloat abstractions, don't clean up dead code... implement a bloated construction over 1000 lines when 100 would do."

These four principles directly address each failure mode.

## The four principles at a glance

<CardGroup cols={2}>
  <Card title="Think Before Coding" icon="brain" href="/principles/think-before-coding">
    State assumptions explicitly. Surface tradeoffs. Ask rather than guess.
  </Card>
  <Card title="Simplicity First" icon="minus" href="/principles/simplicity-first">
    Write the minimum code that solves the problem. Nothing speculative.
  </Card>
  <Card title="Surgical Changes" icon="scalpel" href="/principles/surgical-changes">
    Touch only what you must. Match existing style. Clean up only your own mess.
  </Card>
  <Card title="Goal-Driven Execution" icon="bullseye-arrow" href="/principles/goal-driven-execution">
    Define verifiable success criteria. Transform tasks into testable goals.
  </Card>
</CardGroup>

## Quick install

<Steps>
  <Step title="Add the plugin marketplace">
    In Claude Code, run:
    ```bash
    /plugin marketplace add forrestchang/andrej-karpathy-skills
    ```
  </Step>
  <Step title="Install the plugin">
    ```bash
    /plugin install andrej-karpathy-skills@karpathy-skills
    ```
  </Step>
  <Step title="Start coding">
    The guidelines are now active across all your projects. Claude will surface assumptions, prefer simpler solutions, make surgical edits, and define verifiable goals before implementing.
  </Step>
</Steps>

  Prefer a per-project file? See [Installation](/installation) for the `CLAUDE.md` drop-in option and how to merge with existing project instructions.
